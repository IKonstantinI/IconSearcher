## 🎭 MVP Паттерн

### Что такое MVP?

**Model-View-Presenter** — паттерн для разделения UI логики и бизнес-логики.

### Компоненты

| Компонент | Ответственность | Знает о |
|-----------|-----------------|---------|
| **Model** | Данные и бизнес-правила | Ничего о UI |
| **View** | Отображение и ввод | Только Protocol |
| **Presenter** | Логика отображения, преобразование данных | View Protocol, Model |

### Схема Взаимодействия

```
┌──────────────┐         ┌──────────────┐         ┌──────────────┐
│     View     │────────▶│  Presenter   │────────▶│    Model     │
│  (VC, Cell)  │ Protocol │   (Logic)    │  Data   │   (Struct)   │
│              │◀──────── │              │         │              │
│  showIcons() │  Data    │ viewDidLoad() │         │              │
│  showLoading()│         │ search()     │         │              │
└──────────────┘         └──────────────┘         └──────────────┘
```

### Реализация в IconSearcher

#### Протокол View
```swift
protocol IconSearchViewProtocol: AnyObject {
    func showIcons(viewModels: [IconViewModel])
    func showLoading()
    func hideLoading()
    func showError(title: String, message: String)
}
```

#### Presenter
```swift
final class IconSearchPresenter: IconSearchPresenterProtocol {
    private weak var view: IconSearchViewProtocol?
    private let iconService: IconServiceProtocol
    
    init(view: IconSearchViewProtocol, iconService: IconServiceProtocol) {
        self.view = view
        self.iconService = iconService
    }
    
    func searchButtonTapped(query: String?) {
        // Логика валидации
        // Вызов сервиса
        // Обновление view
    }
}
```

## 📜 Протокольно-Ориентированное Программирование (POP)

### Что такое POP?

**Protocol Oriented Programming** — парадигма, где протоколы являются основой дизайна, а не классы.

### Преимущества

| Преимущество | Описание | Пример |
|--------------|----------|--------|
| **Weak Coupling** | Слабая связанность | View не знает конкретный Presenter |
| **Testability** | Легко тестировать | Mock сервис для тестов |
| **Flexibility** | Гибкость замены | Реальная реализация ↔ Mock |
| **Composition** | Композиция вместо наследования | Протоколы с расширениями |

### Реализация в IconSearcher

#### Все сервисы через протоколы
```swift
// ❌ Плохо: конкретная реализация
final class IconSearchPresenter {
    private let iconifyService: IconifyService
}

// ✅ Хорошо: протокол
final class IconSearchPresenter {
    private let iconService: IconServiceProtocol
}
```

#### Протоколы в проекте
```
IconServiceProtocol
NetworkManagerProtocol
IconSearchViewProtocol
IconSearchPresenterProtocol
IconCellPresenterDelegate
```

## 💉 Внедрение Зависимостей (DI)

### Что такое DI?

**Dependency Injection** — передача зависимостей извне, а не создание внутри класса.

### Типы DI

| Тип | Описание | Пример |
|-----|----------|--------|
| **Initializer** | Через конструктор | `init(service: X)` |
| **Property** | Через свойство | `var service: X?` |
| **Method** | Через метод | `func configure(service: X)` |

### Assembly Pattern

**Assembly** — объект, отвечающий за создание и настройку графа зависимостей.

```swift
final class IconSearchAssembly {
    
    static func assemble() -> IconSearchViewController {
        // 1. Создаём сервисы
        let networkManager = NetworkManager()
        let iconifyService = IconifyService(networkManager: networkManager)
        let imageLoader = ImageLoader()
        
        // 2. Создаём Presenter
        let presenter = IconSearchPresenter(
            view: /* будет VC */,
            iconService: iconifyService
        )
        
        // 3. Создаём ViewController
        let viewController = IconSearchViewController(presenter: presenter)
        
        // 4. Замыкаем цикл (weak reference)
        presenter.view = viewController
        
        return viewController
    }
}
```

## ⚖️ Сравнение Подходов

### MVP vs MVVM

| Критерий | MVP | MVVM |
|----------|-----|------|
| **Связь View-Presenter** | View вызывает Presenter явно | ViewModel не знает о View |
| **Binding** | Ручной вызов методов | Data Binding (Rx, Combine) |
| **Сложность** | Проще для понимания | Сложнее из-за биндингов |
| **Тестирование** | Легко | Легче с Combine |
| **iOS Сообщество** | Меньше популярен | Очень популярен |

**Наш выбор:** MVP — проще для обучения, явно видны связи.

---

### MVC vs MVP

| Критерий | MVC (Apple) | MVP |
|----------|-------------|-----|
| **ViewController** | View + Controller | Только View |
| **Логика** | В ViewController | В Presenter |
| **Тестирование** | Сложно | Легко |
| **Massive VC** | Да | Нет |

**Наш выбор:** MVP — избегаем "Massive ViewController".

---

### VIPER vs MVP

| Критерий | MVP | VIPER |
|----------|-----|-------|
| **Компоненты** | 3 (M-V-P) | 5 (V-I-P-E-R) |
| **Сложность** | Средняя | Высокая |
| **Файлов** | ~10 на модуль | ~20 на модуль |
| **Для проекта** | Идеально | Overengineering |

**Наш выбор:** MVP — баланс между чистотой и сложностью.

## 🎯 CellPresenter Pattern

### Проблема

При активном скролле ячейка уходит в пул и появляется под другим индексом. Картинка не успевает загрузиться → **неправильные данные в ячейке**.

### Решение

```
┌─────────────────────────────────────────────────────┐
│              IconTableViewCell                      │
│  ┌─────────────────────────────────────────────┐   │
│  │           IconCellPresenter                 │   │
│  │  ┌─────────────────────────────────────┐   │   │
│  │  │  - loadImage(url:)                  │   │   │
│  │  │  - cancelLoad()                     │   │   │
│  │  │  - delegate.imageDidLoad(image)     │   │   │
│  │  └─────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### Реализация

```swift
// В ячейке
override func prepareForReuse() {
    super.prepareForReuse()
    cellPresenter.cancelLoad()  // Отменяем загрузку
    imageView.image = nil       // Сбрасываем
}

func configure(with viewModel: IconViewModel, presenter: IconCellPresenter) {
    self.cellPresenter = presenter
    cellPresenter.loadImage(from: viewModel.iconURL)
}
```
