## 🎯 Назначение

### Что Решает

```
Проблема: Прямая работа с URLSession в сервисах
├── Дублирование кода (каждый сервис создаёт свой session)
├── Нет централизованного кэширования
├── Сложно тестировать (нет абстракции)
└── Разная обработка ошибок

Решение: NetworkManager — единый сетевой слой
```

### Зона Ответственности

| Делает | Не Делает |
|--------|-----------|
| ✅ HTTP запросы (GET, POST, etc.) | ❌ Не знает о бизнес-логике |
| ✅ Декодирование JSON | ❌ Не формирует запросы (это сервис) |
| ✅ Кэширование ответов | ❌ Не обрабатывает ошибки UI |
| ✅ Возврат в main thread | ❌ Не решает, когда делать запрос |

## 📜 Протокол NetworkManagerProtocol

### Интерфейс

```swift
import Foundation

/// Протокол сетевого менеджера.
/// 
/// Абстракция для выполнения HTTP запросов.
/// Использует дженерик для автоматического декодирования JSON.
/// 
/// Использование:
/// ```swift
/// networkManager.request(url: url) { (result: Result<MyModel, Error>) in
///     // result.success(MyModel) или result.failure(Error)
/// }
/// ```
protocol NetworkManagerProtocol {
    
    /// Выполнить GET запрос и декодировать ответ.
    /// - Parameters:
    ///   - url: URL для запроса
    ///   - completion: Closure с результатом (Decodable или Error)
    func request<T: Decodable>(
        url: URL,
        completion: @escaping (Result<T, Error>) -> Void
    )
}
```

### Преимущества Протокола

| Преимущество | Описание |
|--------------|----------|
| **Тестируемость** | Mock для unit тестов |
| **Заменяемость** | Можно заменить на Alamofire |
| **Абстракция** | Сервис не знает про URLSession |
| **Дженерик** | Один метод для любых моделей |

## 💻 Реализация NetworkManager

### Production Версия

```swift
import Foundation

/// Менеджер сетевых запросов.
/// 
/// Использует URLSession с URLCache для автоматического кэширования.
/// Все запросы возвращают результат в main thread.
/// 
/// Особенности:
/// - Автоматическое декодирование JSON (Codable)
/// - Кэширование ответов (память + диск)
/// - Политика кэша: returnCacheDataElseLoad
final class NetworkManager: NetworkManagerProtocol {
    
    // MARK: - Properties
    
    /// URLSession для выполнения запросов
    private let session: URLSession
    
    // MARK: - Initialization
    
    /// Инициализация с кастомными параметрами кэша.
    /// - Parameters:
    ///   - memoryCapacity: Размер кэша в памяти (по умолчанию 50 MB)
    ///   - diskCapacity: Размер кэша на диске (по умолчанию 100 MB)
    init(
        memoryCapacity: Int = 50 * 1024 * 1024,
        diskCapacity: Int = 100 * 1024 * 1024
    ) {
        // Конфигурация с кэшированием
        let configuration = URLSessionConfiguration.default
        
        // URLCache для HTTP-ответов
        configuration.urlCache = URLCache(
            memoryCapacity: memoryCapacity,
            diskCapacity: diskCapacity
        )
        
        // Политика кэша: сначала кэш, потом сеть
        configuration.requestCachePolicy = .returnCacheDataElseLoad
        
        // Создаём session
        self.session = URLSession(configuration: configuration)
    }
    
    // MARK: - Public Methods
    
    func request<T: Decodable>(
        url: URL,
        completion: @escaping (Result<T, Error>) -> Void
    ) {
        // Создаём data task
        let task = session.dataTask(with: url) { [weak self] data, response, error in
            guard self != nil else { return }
            
            // Возвращаем результат в main thread
            DispatchQueue.main.async {
                // 1. Проверяем ошибку
                if let error = error {
                    completion(.failure(error))
                    return
                }
                
                // 2. Проверяем данные
                guard let data = data else {
                    completion(.failure(URLError(.cannotParseResponse)))
                    return
                }
                
                // 3. Декодируем JSON
                do {
                    let decoder = JSONDecoder()
                    let decodedObject = try decoder.decode(T.self, from: data)
                    completion(.success(decodedObject))
                } catch {
                    completion(.failure(error))
                }
            }
        }
        
        // Запускаем задачу
        task.resume()
    }
}
```

## 💾 URLCache Настройка

### Что Такое URLCache

**URLCache** — встроенный механизм кэширования HTTP-ответов от Apple.

### Как Работает

```
Запрос → URLCache
         │
         ├─ ✅ Есть в кэше? → Возвращаем (быстро)
         │
         └─ ❌ Нет в кэше? → Сеть → Сохраняем в кэш → Возвращаем
```

### Уровни Кэширования

```
┌─────────────────────────────────────────────────────────────┐
│                    URLCache                                 │
├─────────────────────────────────────────────────────────────┤
│  Memory (50 MB)         │  Disk (100 MB)                   │
│  - Быстрый доступ       │  - Между запусками               │
│  - Сбрасывается         │  - Сохраняется                   │
│  - ~0.1ms               │  - ~10ms                         │
└─────────────────────────────────────────────────────────────┘
```

### Настройки в NetworkManager

```swift
let configuration = URLSessionConfiguration.default

// Кэш в памяти: 50 MB
// Кэш на диске: 100 MB
configuration.urlCache = URLCache(
    memoryCapacity: 50 * 1024 * 1024,
    diskCapacity: 100 * 1024 * 1024
)

// Политика кэширования
configuration.requestCachePolicy = .returnCacheDataElseLoad
```

### Политики Кэширования

| Политика | Описание | Когда |
|----------|----------|-------|
| **useProtocolCachePolicy** | По умолчанию, следует HTTP заголовкам | Для большинства запросов |
| **returnCacheDataElseLoad** | Сначала кэш, потом сеть | Для статических данных |
| **returnCacheDataDontLoad** | Только кэш, без сети | Офлайн-режим |
| **reloadIgnoringLocalCacheData** | Игнорировать кэш | Принудительное обновление |

## ⚠️ Обработка Ошибок

### Типы Ошибок

```swift
// 1. Сетевые ошибки (URLError)
enum URLError: Error {
    case badURL
    case timedOut
    case cannotParseResponse
    case notConnectedToInternet
    // ...
}

// 2. Ошибки декодирования (DecodingError)
enum DecodingError: Error {
    case dataCorrupted
    case typeMismatch
    case keyNotFound
    case valueNotFound
}

// 3. Кастомные ошибки
enum APIError: Error {
    case unauthorized
    case notFound
    case serverError
}
```

### Обработка в NetworkManager

```swift
func request<T: Decodable>(
    url: URL,
    completion: @escaping (Result<T, Error>) -> Void
) {
    let task = session.dataTask(with: url) { data, response, error in
        DispatchQueue.main.async {
            // 1. Сетевая ошибка
            if let error = error {
                completion(.failure(error))
                return
            }
            
            // 2. Нет данных
            guard let data = data else {
                completion(.failure(URLError(.cannotParseResponse)))
                return
            }
            
            // 3. Ошибка декодирования
            do {
                let decodedObject = try JSONDecoder().decode(T.self, from: data)
                completion(.success(decodedObject))
            } catch {
                completion(.failure(error))  // DecodingError
            }
        }
    }
    task.resume()
}
```

### Обработка в Сервисе

```swift
// В IconifyService
func searchIcons(query: String, completion: @escaping (Result<[Icon], Error>) -> Void) {
    networkManager.request(url: url) { (result: Result<SearchResult, Error>) in
        switch result {
        case .failure(let error):
            // Сетевая ошибка или ошибка декодирования
            completion(.failure(error))
            
        case .success(let searchResult):
            // Успех, обрабатываем данные
            completion(.success(searchResult.icons))
        }
    }
}
```

## 🎯 Использование в Сервисах

### Пример: IconifyService

```swift
final class IconifyService {
    
    private let networkManager: NetworkManagerProtocol
    
    init(networkManager: NetworkManagerProtocol = NetworkManager()) {
        self.networkManager = networkManager
    }
    
    func searchIcons(query: String, completion: @escaping (Result<[Icon], Error>) -> Void) {
        // 1. Формируем URL
        var components = URLComponents(string: "https://api.iconify.design/search")
        components?.queryItems = [URLQueryItem(name: "query", value: query)]
        
        guard let url = components?.url else {
            completion(.failure(URLError(.badURL)))
            return
        }
        
        // 2. Выполняем запрос через NetworkManager
        networkManager.request(url: url) { [weak self] (result: Result<SearchResult, Error>) in
            guard let self = self else { return }
            
            switch result {
            case .failure(let error):
                completion(.failure(error))
                
            case .success(let searchResult):
                if searchResult.icons.isEmpty {
                    completion(.success([]))
                    return
                }
                
                // 3. Загружаем детали
                self.fetchDetails(for: searchResult.icons, completion: completion)
            }
        }
    }
}
```

## ✅ Best Practices

### 1. Возврат в Main Thread

```swift
// ✅ Хорошо
DispatchQueue.main.async {
    completion(.success(decodedObject))
}

// ❌ Плохо
completion(.success(decodedObject))  // В background thread!
```

### 2. Weak Self в Closure

```swift
// ✅ Хорошо
session.dataTask(with: url) { [weak self] data, response, error in
    guard self != nil else { return }
    // ...
}

// ❌ Плохо
session.dataTask(with: url) { data, response, error in
    self.handleResponse(data)  // Retain cycle!
}
```

### 3. Дженерик для Любых Моделей

```swift
// ✅ Хорошо: один метод для всех моделей
func request<T: Decodable>(
    url: URL,
    completion: @escaping (Result<T, Error>) -> Void
)

// Использование:
networkManager.request(url: url1) { (result: Result<Model1, Error>) in }
networkManager.request(url: url2) { (result: Result<Model2, Error>) in }

// ❌ Плохо: отдельный метод для каждой модели
func requestModel1(url: URL, completion: ...)
func requestModel2(url: URL, completion: ...)
```

### 4. Протокол для Тестирования

```swift
// ✅ Хорошо
class IconifyService {
    private let networkManager: NetworkManagerProtocol
    
    init(networkManager: NetworkManagerProtocol) {
        self.networkManager = networkManager
    }
}

// В тестах:
let mockNetwork = MockNetworkManager()
let service = IconifyService(networkManager: mockNetwork)

// ❌ Плохо
class IconifyService {
    private let networkManager = NetworkManager()  // Concrete implementation
}
```

## ⚠️ Антипаттерны

### 1. Прямая Работа с URLSession в Сервисе

```swift
// ❌ Плохо
class IconifyService {
    func searchIcons(query: String, completion: ...) {
        // Дублирование кода сети
        URLSession.shared.dataTask(with: url) { ... }
    }
}

// ✅ Хорошо
class IconifyService {
    private let networkManager: NetworkManagerProtocol
    
    func searchIcons(query: String, completion: ...) {
        networkManager.request(url: url, completion: completion)
    }
}
```

### 2. Нет Кэширования

```swift
// ❌ Плохо
let configuration = URLSessionConfiguration.default
// Нет urlCache → каждый запрос в сеть

// ✅ Хорошо
configuration.urlCache = URLCache(
    memoryCapacity: 50 * 1024 * 1024,
    diskCapacity: 100 * 1024 * 1024
)
```

### 3. Игнорирование Ошибок

```swift
// ❌ Плохо
session.dataTask(with: url) { data, _, _ in
    let model = try? JSONDecoder().decode(Model.self, from: data!)
    completion(model)  // Error игнорируется!
}

// ✅ Хорошо
session.dataTask(with: url) { data, _, error in
    if let error = error {
        completion(.failure(error))
        return
    }
    // ...
}
```

### 4. UI Не в Main Thread

```swift
// ❌ Плохо
networkManager.request(url: url) { result in
    // В background thread!
    self.tableView.reloadData()  // Crash!
}

// ✅ Хорошо
networkManager.request(url: url) { result in
    // Уже в main thread (благодаря NetworkManager)
    self.tableView.reloadData()
}
```

## 🧪 Тестирование

### Mock NetworkManager

```swift
import Foundation

/// Mock NetworkManager для unit тестов.
final class MockNetworkManager: NetworkManagerProtocol {
    
    var requestCalled = false
    var lastURL: URL?
    
    var mockResult: Result<Any, Error>?
    
    func request<T: Decodable>(
        url: URL,
        completion: @escaping (Result<T, Error>) -> Void
    ) {
        requestCalled = true
        lastURL = url
        
        if let result = mockResult {
            switch result {
            case .success(let data):
                completion(.success(data as! T))
            case .failure(let error):
                completion(.failure(error))
            }
        }
    }
}
```

### Unit Тест Пример

```swift
import XCTest

final class NetworkManagerTests: XCTestCase {
    
    var networkManager: NetworkManager!
    
    func testRequestSuccess() {
        // Given
        networkManager = NetworkManager()
        let testURL = URL(string: "https://api.iconify.design/search?query=home")!
        
        // When
        let expectation = XCTestExpectation(description: "Request completed")
        networkManager.request(url: testURL) { (result: Result<SearchResult, Error>) in
            // Then
            switch result {
            case .success(let searchResult):
                XCTAssertNotNil(searchResult)
            case .failure:
                XCTFail("Expected success")
            }
            expectation.fulfill()
        }
        
        wait(for: [expectation], timeout: 10.0)
    }
    
    func testRequestInvalidURL() {
        // Given
        networkManager = NetworkManager()
        let testURL = URL(string: "invalid_url")!
        
        // When
        let expectation = XCTestExpectation(description: "Request failed")
        networkManager.request(url: testURL) { (result: Result<SearchResult, Error>) in
            // Then
            if case .failure = result {
                // Ожидаем ошибку
            } else {
                XCTFail("Expected failure")
            }
            expectation.fulfill()
        }
        
        wait(for: [expectation], timeout: 5.0)
    }
}
```
