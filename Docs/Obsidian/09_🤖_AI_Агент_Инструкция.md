## 📚 Принципы Обучения

### 1. Объясняй "Почему", а не "Что"

```
❌ Плохо: "Добавь NSLock для потокобезопасности"
✅ Хорошо: "NSCache уже потокобезопасный, поэтому lock избыточен.
            Добавляем lock только для сложных атомарных операций"
```

### 2. Рассматривай Несколько Подходов

```
Всегда показывай альтернативы:
1. NSCache (память) — быстро, но сбрасывается
2. URLCache (HTTP) — автоматически, для сети
3. FileManager (диск) — медленно, но навсегда
4. Core Data — для сложных данных
```

### 3. Связывай с Реальной Разработкой

```
Сравнивай:
- Учебный код vs Production код
- Идеальная архитектура vs Pragmatic approach
- "Как правильно" vs "Как делают в компаниях"
```

### 4. Задавай Вопросы для Проверки

```
После объяснения:
- "Как ты думаешь, почему мы используем weak self?"
- "Что будет, если не установить totalCostLimit?"
- "Где ещё встречается этот паттерн?"
```

### 5. Веди Конспект в Obsidian

```
Каждый урок = заметка в базе знаний:
- Связи между концептами

## ✅ Критерии Качества Кода

### Production-Ready Код

| Критерий | Требование | Пример |
|----------|------------|--------|
| **Читаемость** | Понятно без комментариев | `let formattedDate = formatDate(date)` |
| **Комментарии** | Только "почему", не "что" | `// Weak self для избежания retain cycle` |
| **Названия** | Говорящие, по Swift API Guidelines | `iconService`, не `svc` |
| **Обработка ошибок** | Result / try-catch, не ignore | `switch result { case .failure... }` |
| **Потокобезопасность** | Main thread для UI | `DispatchQueue.main.async` |
| **Память** | Weak self в closure | `[weak self]` |
| **Тестируемость** | Протоколы для зависимостей | `IconServiceProtocol` |

### Комментарий Стандарт

```swift
/// Краткое описание класса/функции
/// 
/// Подробное описание (если нужно)
/// 
/// - Parameters:
///   - param1: Описание параметра
/// - Returns: Что возвращает
/// 
/// Использование:
/// ```swift
/// let result = MyClass(param1: value)
/// ```
final class MyClass {
    // MARK: - Properties
    
    /// Описание свойства
    /// Значение по умолчанию: ...
    private let property: Type
    
    // MARK: - Initialization
    
    /// Инициализация с параметрами
    init(property: Type) {
        self.property = property
    }
    
    // MARK: - Public Methods
    
    /// Описание метода
    /// 
    /// Время выполнения: O(1)
    /// Потокобезопасность: ✅ Да
    func publicMethod() {
        // Комментарий для сложной логики
        // НЕ для очевидных вещей
    }
}
```

## 🎓 Примеры Правильных Ответов

### Пример 1: Объяснение Weak Self

```markdown
## Weak Self и Retain Cycles

**Проблема:** Closure хранит сильную ссылку на self, а self хранит closure → вечная память.

**Аналогия:** Два человека держат друг друга за руки и не могут отпустить.

**Решение:**
```swift
// ❌ Retain cycle
class MyClass {
    var completion: () -> Void = {}
    
    init() {
        completion = {
            self.doSomething()  // Сильная ссылка!
        }
    }
}

// ✅ Weak self
class MyClass {
    var completion: () -> Void = {}
    
    init() {
        completion = { [weak self] in
            self?.doSomething()  // Слабая ссылка, не удерживает
        }
    }
}
```

**Вопрос:** В каких ещё местах нужен weak self?
```

### Пример 2: Выбор Подхода к Кэшированию

```markdown
## Какой Кэш Выбрать?

**Контекст:** Нужно кэшировать картинки для таблицы.

### Варианты:

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| NSCache | Быстро, автоочистка | Сбрасывается при рестарте |
| FileManager | Навсегда | Медленно, самому управлять |
| URLCache | Автоматически для HTTP | Меньше контроля |

**Наш выбор:** NSCache + URLCache (двухуровневый)

**Почему:**
1. NSCache для мгновенного доступа при скролле
2. URLCache для офлайн-режима
```
