## 🎯 Назначение

### Что Решает

```
Проблема: API Iconify требует двухэтапной загрузки
├── 1. Поиск иконок → список имён
├── 2. Детали иконок → метаданные (теги, размеры)
└── Нужно объединить в один ответ для клиента

Решение: IconifyService скрывает сложность API
```

### Зона Ответственности

| Делает | Не Делает |
|--------|-----------|
| ✅ Поиск иконок по запросу | ❌ Не знает о UI (ViewModel) |
| ✅ Загрузка деталей из коллекции | ❌ Не кэширует картинки (это ImageLoader) |
| ✅ Объединение данных в [Icon] | ❌ Не решает, когда искать |
| ✅ Обработка ошибок API | ❌ Не отображает ошибки (это Presenter) |

## 🌐 API Iconify

### Endpoints

#### 1. Поиск Иконок

```
GET https://api.iconify.design/search?query={query}
```

**Пример:**
```
GET https://api.iconify.design/search?query=home
```

**Ответ:**
```json
{
  "icons": ["mdi:home", "fa:home", "bi:house"],
  "total": 3,
  "collections": {
    "mdi": { "name": "Material Design Icons" },
    "fa": { "name": "Font Awesome" },
    "bi": { "name": "Bootstrap Icons" }
  }
}
```

**Модель:** `SearchResult`

---

#### 2. Детали Коллекции

```
GET https://api.iconify.design/{prefix}.json?icons={icons}
```

**Пример:**
```
GET https://api.iconify.design/mdi.json?icons=home,user,settings
```

**Ответ:**
```json
{
  "prefix": "mdi",
  "width": 24,
  "height": 24,
  "icons": {
    "home": {
      "tags": ["house", "building"],
      "body": "M10 20v-6h4v6h5v-8h3L12 3 2 12h3v8z"
    },
    "user": {
      "tags": ["person", "account"],
      "body": "M12 12c2.21 0 4-1.79 4-4s-1.79-4-4-4-4 4 1.79 4 4 4z..."
    }
  }
}
```

**Модель:** `IconDetailResponse`

---

#### 3. Изображение Иконки

```
GET https://api.iconify.design/{prefix}/{icon}.svg
GET https://api.iconify.design/{prefix}/{icon}.png
```

**Пример:**
```
GET https://api.iconify.design/mdi/home.svg
```

**Ответ:** SVG или PNG изображение

## 📜 Протокол IconServiceProtocol

### Интерфейс

```swift
import Foundation

/// Протокол сервиса поиска иконок.
/// 
/// Абстракция для поиска и загрузки данных об иконках.
/// Скрывает детали работы с API Iconify.
/// 
/// Использование:
/// ```swift
/// iconService.searchIcons(query: "home") { result in
///     case .success(let icons): // [Icon]
///     case .failure(let error): // Error
/// }
/// ```
protocol IconServiceProtocol {
    
    /// Поиск иконок по запросу.
    /// - Parameters:
    ///   - query: Поисковый запрос
    ///   - completion: Closure с результатом ([Icon] или Error)
    func searchIcons(
        query: String,
        completion: @escaping (Result<[Icon], Error>) -> Void
    )
}
```

## 💻 Реализация IconifyService

### Production Версия

```swift
import Foundation

/// Сервис для работы с API Iconify.
/// 
/// Реализует двухэтапную загрузку:
/// 1. Поиск иконок → список имён
/// 2. Загрузка деталей → метаданные (теги, размеры)
/// 
/// Особенности:
/// - Группировка по коллекциям для оптимизации запросов
/// - Параллельная загрузка деталей
/// - Агрегация результатов в единый массив
final class IconifyService {
    
    // MARK: - Properties
    
    /// Менеджер сетевых запросов
    private let networkManager: NetworkManagerProtocol
    
    // MARK: - Initialization
    
    /// Инициализация с network manager.
    /// - Parameter networkManager: Сетевой менеджер (по умолчанию NetworkManager())
    init(networkManager: NetworkManagerProtocol = NetworkManager()) {
        self.networkManager = networkManager
    }
    
    // MARK: - Public Methods
    
    func searchIcons(
        query: String,
        completion: @escaping (Result<[Icon], Error>) -> Void
    ) {
        // 1. Формируем URL для поиска
        var components = URLComponents(string: "https://api.iconify.design/search")
        components?.queryItems = [URLQueryItem(name: "query", value: query)]
        
        guard let url = components?.url else {
            completion(.failure(URLError(.badURL)))
            return
        }
        
        // 2. Выполняем поиск
        networkManager.request(url: url) { [weak self] (result: Result<SearchResult, Error>) in
            guard let self = self else { return }
            
            switch result {
            case .failure(let error):
                completion(.failure(error))
                
            case .success(let searchResult):
                // Пустой результат
                if searchResult.icons.isEmpty {
                    completion(.success([]))
                    return
                }
                
                // 3. Загружаем детали
                self.fetchDetails(for: searchResult.icons, completion: completion)
            }
        }
    }
    
    // MARK: - Private Methods
    
    /// Загрузить детали для списка иконок.
    /// 
    /// Алгоритм:
    /// 1. Группируем иконки по префиксу коллекции
    /// 2. Для каждой коллекции делаем один запрос
    /// 3. Параллельно загружаем все коллекции
    /// 4. Объединяем результаты
    private func fetchDetails(
        for iconNames: [String],
        completion: @escaping (Result<[Icon], Error>) -> Void
    ) {
        // 1. Создаём карту иконок
        var iconsMap: [String: Icon] = [:]
        iconNames.forEach { iconsMap[$0] = Icon(fullName: $0) }
        
        // 2. Группируем по префиксу
        let groupedByPrefix = Dictionary(
            grouping: iconNames,
            by: { Icon(fullName: $0).collectionPrefix }
        )
        
        // 3. DispatchGroup для параллельной загрузки
        let group = DispatchGroup()
        
        // 4. Сохраняем первую ошибку (если будет)
        var firstError: Error?
        let errorLock = NSLock()
        
        // 5. Для каждой коллекции
        for (prefix, names) in groupedByPrefix {
            let iconNamesOnly = names.map { Icon(fullName: $0).iconName }
            
            // Формируем URL: /{prefix}.json?icons={icons}
            var components = URLComponents(
                string: "https://api.iconify.design/\(prefix).json"
            )
            components?.queryItems = [
                URLQueryItem(name: "icons", value: iconNamesOnly.joined(separator: ":"))
            ]
            
            guard let url = components?.url else { continue }
            
            // 6. Запускаем запрос
            group.enter()
            networkManager.request(url: url) { (result: Result<IconDetailResponse, Error>) in
                switch result {
                case .success(let detailResponse):
                    // 7. Обновляем данные иконок
                    for (iconName, metaData) in detailResponse.icons {
                        let fullName = "\(prefix):\(iconName)"
                        iconsMap[fullName]?.width = detailResponse.width
                        iconsMap[fullName]?.height = detailResponse.height
                        iconsMap[fullName]?.tags = metaData.tags ?? []
                    }
                    
                case .failure(let error):
                    // 8. Сохраняем ошибку (первую)
                    errorLock.lock()
                    if firstError == nil {
                        firstError = error
                    }
                    errorLock.unlock()
                }
                
                group.leave()
            }
        }
        
        // 9. Ждём завершения всех запросов
        group.notify(queue: .main) {
            if let error = firstError {
                completion(.failure(error))
            } else {
                // 10. Возвращаем результаты в исходном порядке
                let finalIcons = iconNames.compactMap { iconsMap[$0] }
                completion(.success(finalIcons))
            }
        }
    }
}
```

## 🔄 Двухэтапная Загрузка

### Детальная Схема

```
Этап 1: Поиск
┌─────────────────────────────────────────────────────────────┐
│ GET /search?query=home                                      │
│                                                             │
│ Response:                                                   │
│ {                                                           │
│   "icons": ["mdi:home", "fa:home", "bi:house"],            │
│   "total": 3,                                               │
│   "collections": {...}                                      │
│ }                                                           │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
                    ["mdi:home", "fa:home", "bi:house"]
                            │
                            ▼
Этап 2: Группировка
┌─────────────────────────────────────────────────────────────┐
│ {                                                           │
│   "mdi": ["mdi:home"],                                      │
│   "fa": ["fa:home"],                                        │
│   "bi": ["bi:house"]                                        │
│ }                                                           │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
Этап 3: Параллельные Запросы
┌─────────────────────────────────────────────────────────────┐
│ GET /mdi.json?icons=home  ──┐                               │
│ GET /fa.json?icons=home   ──┼──> Параллельно                │
│ GET /bi.json?icons=house  ──┘                               │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
Этап 4: Объединение
┌─────────────────────────────────────────────────────────────┐
│ Icon(                                                       │
│   fullName: "mdi:home",                                     │
│   tags: ["house", "building"],                              │
│   width: 24, height: 24                                     │
│ )                                                           │
│ ...                                                         │
└─────────────────────────────────────────────────────────────┘
```

### Почему Группировка Важна

**Без группировки:**
```
["mdi:home", "mdi:user", "mdi:settings"]
3 иконки → 3 запроса к API
```

**С группировкой:**
```
["mdi:home", "mdi:user", "mdi:settings"]
3 иконки → 1 запрос: /mdi.json?icons=home:user:settings
```

**Выгода:** В 3 раза меньше запросов!

## ⚠️ Обработка Ошибок

### Типы Ошибок

```swift
// 1. Ошибки URL
URLError.badURL

// 2. Сетевые ошибки
URLError.notConnectedToInternet
URLError.timedOut

// 3. Ошибки декодирования
DecodingError.dataCorrupted
DecodingError.keyNotFound

// 4. Ошибки API (HTTP статус)
// Iconify обычно возвращает 200 или ошибку сети
```

### Стратегия Обработки

```swift
// В searchIcons
switch result {
case .failure(let error):
    // Сетевая ошибка → сразу возвращаем
    completion(.failure(error))
    
case .success(let searchResult):
    if searchResult.icons.isEmpty {
        // Пустой результат → не ошибка, просто []
        completion(.success([]))
        return
    }
    
    // Загружаем детали
    fetchDetails(for: searchResult.icons) { result in
        switch result {
        case .failure(let error):
            // Ошибка при загрузке деталей
            completion(.failure(error))
            
        case .success(let icons):
            // Успех
            completion(.success(icons))
        }
    }
}
```

### Частичные Ошибки

**Ситуация:** 3 коллекции, 1 ошибка при загрузке.

**Решение:**
```swift
// Сохраняем первую ошибку
var firstError: Error?
let errorLock = NSLock()

// В closure каждого запроса
case .failure(let error):
    errorLock.lock()
    if firstError == nil {
        firstError = error  // Первая ошибка
    }
    errorLock.unlock()
    // Не прерываем, другие запросы продолжаются

// В group.notify
if let error = firstError {
    completion(.failure(error))  // Возвращаем первую ошибку
} else {
    completion(.success(finalIcons))  // Или успех
}
```

## ✅ Best Practices

### 1. Протокол для Абстракции

```swift
// ✅ Хорошо
protocol IconServiceProtocol {
    func searchIcons(query: String, completion: ...)
}

class IconifyService: IconServiceProtocol { }

// В Presenter:
private let iconService: IconServiceProtocol

// ❌ Плохо
private let iconService: IconifyService  // Concrete implementation
```

### 2. Weak Self в Closure

```swift
// ✅ Хорошо
networkManager.request(url: url) { [weak self] result in
    guard let self = self else { return }
    self.fetchDetails(...)
}

// ❌ Плохо
networkManager.request(url: url) { result in
    self.fetchDetails(...)  // Retain cycle!
}
```

### 3. Возврат в Main Thread

```swift
// ✅ Хорошо (NetworkManager уже возвращает в main)
group.notify(queue: .main) {
    completion(.success(finalIcons))
}

// ❌ Плохо
group.notify(queue: .global()) {
    completion(.success(finalIcons))  // В background!
}
```

### 4. Сохранение Порядка

```swift
// ✅ Хорошо: сохраняем исходный порядок
let finalIcons = iconNames.compactMap { iconsMap[$0] }

// ❌ Плохо: порядок теряется
let finalIcons = iconsMap.values  // Порядок не гарантирован!
```

## ⚠️ Антипаттерны

### 1. Один Запрос на Иконку

```swift
// ❌ Плохо: N запросов для N иконок
for iconName in iconNames {
    networkManager.request(url: "...") { ... }
}

// ✅ Хорошо: группировка по коллекции
let grouped = Dictionary(grouping: iconNames, by: { ... })
for (prefix, names) in grouped {
    // Один запрос на коллекцию
}
```

### 2. Игнорирование Пустого Результата

```swift
// ❌ Плохо
networkManager.request(url: url) { result in
    switch result {
    case .success(let searchResult):
        self.fetchDetails(for: searchResult.icons)  // Может быть []!
    }
}

// ✅ Хорошо
case .success(let searchResult):
    if searchResult.icons.isEmpty {
        completion(.success([]))
        return
    }
    self.fetchDetails(...)
```

### 3. Нет Lock для Shared State

```swift
// ❌ Плохо
var firstError: Error?

// В closure (параллельно!)
case .failure(let error):
    if firstError == nil {  // Race condition!
        firstError = error
    }

// ✅ Хорошо
var firstError: Error?
let errorLock = NSLock()

case .failure(let error):
    errorLock.lock()
    if firstError == nil {
        firstError = error
    }
    errorLock.unlock()
```

## 🔄 Альтернативы

### Iconfinder API (Оригинал из ТЗ)

**URL:** `https://api.iconfinder.com/v4/`

**Плюсы:**
- ✅ Больше иконок (4M+)
- ✅ Фильтры (бесплатные, premium, цвет)
- ✅ Официальное ТЗ

**Минусы:**
- ❌ Требуется API key
- ❌ Лимит: 1000 запросов/месяц (бесплатно)
- ❌ Сложнее авторизация

**Пример запроса:**
```
GET https://api.iconfinder.com/v4/icons/search
    ?query=home
    &count=20
    &offset=0
    &premium=0
    &types=vector,bitmap
Authorization: Bearer {API_KEY}
```

---

### IconScout API

**URL:** `https://api.iconscout.com/v3/`

**Плюсы:**
- ✅ Качественные иконки
- ✅ Есть бесплатные

**Минусы:**
- ❌ Требуется API key
- ❌ Меньше бесплатных

---

### Iconify (Наш Выбор)

**Плюсы:**
- ✅ Не требует API key
- ✅ Безлимитные запросы
- ✅ 100K+ иконок
- ✅ Простой API

**Минусы:**
- ❌ Меньше метаданных
- ❌ Нет фильтрации по лицензиям

### Unit Тест Пример

```swift
import XCTest

final class IconifyServiceTests: XCTestCase {
    
    var service: IconifyService!
    var mockNetwork: MockNetworkManager!
    
    func testSearchIconsSuccess() {
        // Given
        mockNetwork = MockNetworkManager()
        mockNetwork.mockResult = Result.success(
            SearchResult(icons: ["mdi:home"], total: 1, collections: [:])
