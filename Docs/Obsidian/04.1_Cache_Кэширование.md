## 🎯 Зачем Нужно Кэширование

### Проблемы Без Кэша

```
Пользователь скроллит таблицу с иконками
         │
         ▼
┌─────────────────────────────────────────┐
│  Ячейка 1: Запрос картинки → Сеть      │ ← 500ms
│  Ячейка 2: Запрос картинки → Сеть      │ ← 500ms
│  Ячейка 3: Запрос картинки → Сеть      │ ← 500ms
│  ...                                    │
│  Ячейка 1 ушла в пул                    │
│  Пользователь скроллит обратно          │
│  Ячейка 1: Запрос картинки → Сеть      │ ← 500ms (опять!)
└─────────────────────────────────────────┘
```

**Результат:**
- ❌ Медленная прокрутка
- ❌ Лишние запросы к API
- ❌ Перерасход трафика
- ❌ Возможные баны от API

### Преимущества Кэша

| Преимущество | Описание |
|--------------|----------|
| **Скорость** | Доступ из памяти: ~0.1ms vs сеть: ~500ms |
| **Трафик** | Меньше запросов = экономия |
| **UX** | Плавный скролл без лагов |
| **Офлайн** | Повторный просмотр работает без сети |

## 📚 Подходы к Кэшированию в iOS

### 1. NSCache (Память)

**Документация:** [NSCache | Apple](https://developer.apple.com/documentation/foundation/nscache)

| Характеристика | Значение |
|----------------|----------|
| **Хранение** | Оперативная память |
| **Сохранение** | Только во время работы приложения |
| **Потокобезопасность** | ✅ Да |
| **Автоматическая очистка** | ✅ При memory warning |
| **Скорость доступа** | ⚡⚡⚡ Очень быстро |

**Когда использовать:**
- Кэширование изображений при скролле
- Временные данные сессии
- Часто используемые объекты

**Пример:**
```swift
let cache = NSCache<NSString, UIImage>()
cache.setObject(image, forKey: "icon_key")
let image = cache.object(forKey: "icon_key")
```

---

### 2. URLCache (HTTP)

**Документация:** [URLCache | Apple](https://developer.apple.com/documentation/foundation/urlcache)

| Характеристика | Значение |
|----------------|----------|
| **Хранение** | Память + Диск |
| **Сохранение** | ✅ Между запусками |
| **Потокобезопасность** | ✅ Да |
| **Учитывает HTTP заголовки** | ✅ Cache-Control, ETag |
| **Скорость доступа** | ⚡⚡ Быстро (память) / ⚡ Медленно (диск) |

**Когда использовать:**
- Кэширование ответов API
- Статические ресурсы (картинки, JSON)
- Офлайн-режим

**Пример из NetworkManager:**
```swift
let configuration = URLSessionConfiguration.default
configuration.urlCache = URLCache(
    memoryCapacity: 50 * 1024 * 1024,  // 50 MB
    diskCapacity: 100 * 1024 * 1024    // 100 MB
)
```

---

### 3. FileManager (Диск)

**Документация:** [FileManager | Apple](https://developer.apple.com/documentation/foundation/filemanager)

| Характеристика | Значение |
|----------------|----------|
| **Хранение** | Файловая система |
| **Сохранение** | ✅ Навсегда |
| **Потокобезопасность** | ❌ Нет (нужен lock) |
| **Контроль размера** | ✅ Полный |
| **Скорость доступа** | ⚡ Медленно |

**Когда использовать:**
- Большие файлы (изображения > 1MB)
- Документы пользователя
- Офлайн-база данных

**Пример:**
```swift
let fileManager = FileManager.default
let url = cacheDirectory.appendingPathComponent("image.png")
try data.write(to: url)  // Сохранение
let data = try Data(contentsOf: url)  // Чтение
```

### 4. Core Data (Структурированные данные)

**Документация:** [Core Data | Apple](https://developer.apple.com/documentation/coredata)

| Характеристика | Значение |
|----------------|----------|
| **Хранение** | SQLite база |
| **Сохранение** | ✅ Навсегда |
| **Запросы** | ✅ Мощные (фильтрация, сортировка) |
| **Связи** | ✅ Между объектами |
| **Скорость доступа** | ⚡⚡ Средне |

**Когда использовать:**
- Сущности с отношениями (пользователи, сообщения)
- Сложные запросы с фильтрацией
- История действий

---

### 5. UserDefaults (Настройки)

**Документация:** [UserDefaults | Apple](https://developer.apple.com/documentation/foundation/userdefaults)

| Характеристика | Значение |
|----------------|----------|
| **Хранение** | Plist файл |
| **Сохранение** | ✅ Навсегда |
| **Типы данных** | Только примитивы |
| **Скорость записи** | ⚠️ Блокирует поток |

**⚠️ НЕ для кэширования!** Только для настроек.

## 🔍 NSCache Детально

### Почему NSCache Лучше Dictionary

| Характеристика | Dictionary | NSCache |
|----------------|------------|---------|
| **Потокобезопасность** | ❌ Нет | ✅ Да |
| **Автоочистка при memory warning** | ❌ Нет | ✅ Да |
| **Лимиты (count/cost)** | ❌ Нет | ✅ Да |
| **Копирование объектов** | ✅ Да | ❌ Нет (быстрее) |

### Ключевые Параметры

```swift
let cache = NSCache<NSString, UIImage>()

// Максимальное количество объектов
cache.countLimit = 100

// Максимальный объём памяти (в байтах)
cache.totalCostLimit = 50 * 1024 * 1024  // 50 MB

// Для UIImage можно установить cost = размер в байтах
cache.setObject(image, forKey: "key", cost: imageData.count)
```

### Почему totalCostLimit Важен

**Без ограничения:**
```
Кэш растёт → Память заканчивается → Memory Warning → Crash
```

**С ограничением:**
```
Кэш растёт → Достиг лимита → NSCache удаляет старые объекты → Работаем
```

## 💻 Реализация Cache.swift

### Production Версия (без избыточного NSLock)

```swift
import Foundation

/// Универсальный кэш для хранения объектов в памяти.
/// Потокобезопасен благодаря NSCache.
/// 
/// Использование:
/// ```swift
/// let cache = Cache<URL, UIImage>()
/// cache.totalCostLimit = 50 * 1024 * 1024  // 50 MB
/// cache.countLimit = 100
/// cache.setValue(image, forKey: url)
/// let image = cache.value(forKey: url)
/// ```
final class Cache<Key: Hashable, Value> {
    
    // MARK: - Properties
    
    /// NSCache — потокобезопасная коллекция от Apple
    /// Автоматически освобождает память при нехватке
    private let cache = NSCache<NSObject, AnyObject>()
    
    // MARK: - Configuration
    
    /// Максимальное количество элементов в кэше
    /// Пример: cache.countLimit = 100
    var countLimit: Int {
        get { cache.countLimit }
        set { cache.countLimit = newValue }
    }
    
    /// Максимальный объём памяти в байтах
    /// Пример: cache.totalCostLimit = 50 * 1024 * 1024 (50 MB)
    var totalCostLimit: Int {
        get { cache.totalCostLimit }
        set { cache.totalCostLimit = newValue }
    }
    
    // MARK: - Public Methods
    
    /// Получить значение из кэша по ключу
    /// - Parameter key: Ключ для поиска
    /// - Returns: Значение, если найдено, иначе nil
    /// 
    /// Время выполнения: O(1)
    func value(forKey key: Key) -> Value? {
        // NSCache уже потокобезопасный — lock не нужен
        return cache.object(forKey: key as NSObject) as? Value
    }
    
    /// Сохранить значение в кэш
    /// - Parameters:
    ///   - value: Значение для сохранения (если nil — удалит старый ключ)
    ///   - key: Ключ для доступа к значению
    /// 
    /// Время выполнения: O(1)
    func setValue(_ value: Value?, forKey key: Key) {
        if let value = value {
            cache.setObject(value as AnyObject, forKey: key as NSObject)
        } else {
            cache.removeObject(forKey: key as NSObject)
        }
    }
    
    /// Удалить значение из кэша по ключу
    /// - Parameter key: Ключ для удаления
    func removeValue(forKey key: Key) {
        cache.removeObject(forKey: key as NSObject)
    }
    
    /// Очистить весь кэш
    /// Используется при выходе из аккаунта или критических ошибках
    func removeAll() {
        cache.removeAllObjects()
    }
}
```

## ✅ Best Practices

### 1. Правильный Выбор Limits

```swift
// ✅ Хорошо: разумные ограничения
let imageCache = Cache<URL, UIImage>()
imageCache.totalCostLimit = 50 * 1024 * 1024  // 50 MB
imageCache.countLimit = 100

// ❌ Плохо: нет ограничений
let imageCache = Cache<URL, UIImage>()  // Может вызвать crash
```

### 2. Weak Self в Closure

```swift
// ✅ Хорошо
service.loadData { [weak self] result in
    self?.updateUI()
}

// ❌ Плохо: retain cycle
service.loadData {
    self.updateUI()  // Сильная ссылка на self
}
```

### 3. Главный Поток для UI

```swift
// ✅ Хорошо
imageLoader.load(url: url) { [weak self] image in
    DispatchQueue.main.async {
        self?.imageView.image = image
    }
}

// ❌ Плохо: обновление UI не в main thread
imageLoader.load(url: url) { image in
    self.imageView.image = image  // Может вызвать crash
}
```

### 4. Отмена Загрузки при Reuse

```swift
// В UITableViewCell
override func prepareForReuse() {
    super.prepareForReuse()
    cellPresenter.cancelLoad()  // Отменяем загрузку
    imageView.image = nil       // Сбрасываем изображение
}
```

## ⚠️ Антипаттерны

### 1. UserDefaults для Кэша

```swift
// ❌ КАТЕГОРИЧЕСКИ НЕЛЬЗЯ
UserDefaults.standard.set(imageData, forKey: "cached_image")

// Почему плохо:
// - Блокирует главный поток при записи
// - Пишет на диск каждый раз
// - Нет лимитов размера
```

### 2. Dictionary Вместо NSCache

```swift
// ❌ Плохо
var cache: [URL: UIImage] = [:]

// Проблемы:
// - Не потокобезопасный
// - Нет автоочистки при memory warning
// - Копирует объекты (медленнее)
```

### 3. Нет Ограничений

```swift
// ❌ Плохо
let cache = NSCache<NSString, UIImage>()
// Нет countLimit и totalCostLimit

// Результат:
// - Кэш растёт бесконечно
// - Memory warning
// - Crash приложения
```

### 4. Избыточный Lock

```swift
// ❌ Избыточно (NSCache уже потокобезопасный)
private let lock = NSLock()

func setValue(_ value: Value, forKey key: Key) {
    lock.lock()  // Не нужно!
    cache.setObject(value, forKey: key)
    lock.unlock()
}

// ✅ Хорошо: без lock для простых операций
func setValue(_ value: Value, forKey key: Key) {
    cache.setObject(value, forKey: key)
}
```

## 🧪 Тестирование Кэша

### Unit Тест Пример

```swift
import XCTest

final class CacheTests: XCTestCase {
    
    var cache: Cache<String, UIImage>!
    
    override func setUp() {
        cache = Cache()
        cache.countLimit = 10
    }
    
    func testSetValueAndGetValue() {
        // Given
        let image = UIImage()
        let key = "test_key"
        
        // When
        cache.setValue(image, forKey: key)
        let retrieved = cache.value(forKey: key)
        
        // Then
        XCTAssertNotNil(retrieved)
        XCTAssertEqual(retrieved, image)
    }
    
    func testRemoveValue() {
        // Given
        cache.setValue(UIImage(), forKey: "key")
        
        // When
        cache.removeValue(forKey: "key")
        
        // Then
        XCTAssertNil(cache.value(forKey: "key"))
    }
    
    func testRemoveAll() {
        // Given
        cache.setValue(UIImage(), forKey: "key1")
        cache.setValue(UIImage(), forKey: "key2")
        
        // When
        cache.removeAll()
        
        // Then
        XCTAssertNil(cache.value(forKey: "key1"))
        XCTAssertNil(cache.value(forKey: "key2"))
    }
}
```
